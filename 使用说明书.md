# MobileDB Server 使用说明书

## 1. 系统概述

MobileDB Server 是一款运行于 Android 设备上的轻量级数据库服务器软件。它将 Android 手机/平板转化为一个支持 TCP 和 HTTP 协议的远程数据库服务器，适用于移动办公、边缘计算、临时组网等场景。

### 1.1 系统架构

### 1.2 技术规格

| 项目 | 内容 |
|------|------|
| 最低系统版本 | Android 10 (API 29) |
| 网络协议 | TCP (Netty) + HTTP/REST (Ktor) + WebSocket |
| 数据库引擎 | SQLite + WAL 模式 |
| 缓存引擎 | Caffeine (内存 LRU 缓存) |
| 认证方式 | JWT (HS256) + 用户名/密码 |
| 传输加密 | TLS 1.2+ (可选) |
| 存储加密 | AES-256-GCM (可选) |
| 依赖注入 | Hilt |
| TCP 默认端口 | 9090 |
| HTTP 默认端口 | 8080 |

## 2. 安装与部署

### 2.1 环境要求

- **操作系统**: Android 10 或更高版本
- **硬件最低配置**: CPU 4核1.8GHz, RAM 2GB, 可用存储 1GB
- **推荐配置**: CPU 8核2.0GHz+, RAM 4GB+, eMMC 5.1+ 存储
- **网络**: 稳定的局域网 IP（WiFi 或有线网络）

### 2.2 安装步骤

1. 下载 APK 安装包
2. 在 Android 设备上开启「允许安装未知来源应用」
3. 点击 APK 文件安装
4. 安装完成后，点击「打开」启动应用

### 2.3 权限说明

应用需要以下权限：

| 权限 | 用途 |
|------|------|
| `INTERNET` | 提供网络服务 |
| `ACCESS_NETWORK_STATE` | 监听网络变化 |
| `ACCESS_WIFI_STATE` | 获取 WiFi IP 地址 |
| `FOREGROUND_SERVICE` | 后台运行服务 |
| `RECEIVE_BOOT_COMPLETED` | 开机自启 |
| `WAKE_LOCK` | 防止设备休眠 |
| `POST_NOTIFICATIONS` | 发送告警通知 |

## 5. TCP 协议说明

### 5.1 协议格式

TCP 协议采用自定义二进制格式，基于 JSON 序列化：

```
[4字节消息体长度][消息体JSON]
```

- **长度字段**: 4 字节大端整数，表示消息体的字节长度
- **消息体**: JSON 格式的请求/响应数据

### 5.2 消息类型

| 类型 | 值 | 方向 | 描述 |
|------|-----|------|------|
| AUTH | 1 | 客户端→服务器 | 认证请求 |
| INSERT | 2 | 客户端→服务器 | 插入文档 |
| GET | 3 | 客户端→服务器 | 获取文档 |
| QUERY | 4 | 客户端→服务器 | 查询集合 |
| UPDATE | 5 | 客户端→服务器 | 更新文档 |
| DELETE | 6 | 客户端→服务器 | 删除文档 |
| CREATE_DB | 7 | 客户端→服务器 | 创建数据库 |
| ERROR | 255 | 服务器→客户端 | 错误响应 |

### 5.3 请求示例

**认证请求：**
```json
{
    "type": "AUTH",
    "params": {
        "username": "admin",
        "password": "admin123"
    }
}
```

**插入文档：**
```json
{
    "type": "INSERT",
    "params": {
        "db": "mydb",
        "collection": "users",
        "key": "user001",
        "value": {"name": "张三", "age": 28}
    }
}
```

**查询文档：**
```json
{
    "type": "GET",
    "params": {
        "db": "mydb",
        "collection": "users",
        "key": "user001"
    }
}
```

### 5.4 响应格式

**成功响应：**
```json
{
    "type": "GET",
    "data": {
        "key": "user001",
        "value": {"name": "张三", "age": 28}
    }
}
```

**错误响应：**
```json
{
    "type": "ERROR",
    "data": {
        "message": "Document not found"
    }
}
```

### 5.5 TCP 客户端示例 (Python)

```python
import socket
import json
import struct

class MobileDBClient:
    def __init__(self, host, port=9090):
        self.host = host
        self.port = port
        self.sock = None

    def connect(self):
        self.sock = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        self.sock.connect((self.host, self.port))

    def send_request(self, msg_type, params):
        body = json.dumps({"type": msg_type, "params": params})
        header = struct.pack("!I", len(body))
        self.sock.sendall(header + body.encode())

    def receive_response(self):
        header = self.sock.recv(4)
        length = struct.unpack("!I", header)[0]
        data = b""
        while len(data) < length:
            data += self.sock.recv(length - len(data))
        return json.loads(data.decode())

    def authenticate(self, username, password):
        self.send_request("AUTH", {"username": username, "password": password})
        return self.receive_response()

    def get(self, db, collection, key):
        self.send_request("GET", {"db": db, "collection": collection, "key": key})
        return self.receive_response()

    def insert(self, db, collection, key, value):
        self.send_request("INSERT", {"db": db, "collection": collection, "key": key, "value": value})
        return self.receive_response()

    def close(self):
        if self.sock:
            self.sock.close()

# 使用示例
client = MobileDBClient("192.168.1.100", 9090)
client.connect()
result = client.authenticate("admin", "admin123")
print(result)
client.insert("mydb", "users", "user1", {"name": "Alice"})
data = client.get("mydb", "users", "user1")
print(data)
client.close()
```

## 7. 监控与告警

### 7.1 实时监控指标

系统自动收集以下指标（每秒采集）：

| 指标 | 描述 | 告警阈值 |
|------|------|---------|
| CPU 使用率 | 当前 CPU 占用百分比 | > 80% |
| 内存使用率 | 当前内存占用百分比 | > 90% |
| 存储剩余 | 设备可用存储空间 | < 200MB |
| 活跃连接数 | 当前连接数 | > 150 |
| QPS | 每秒查询数 | — |
| 网络上行 | 发送字节数 | — |
| 网络下行 | 接收字节数 | — |

### 7.2 告警规则

当监控指标超过阈值时，系统会：
1. 在应用通知栏推送告警通知
2. 在 AlertManager 中记录告警历史
3. 同一指标告警冷却期为 5 分钟，避免重复通知

### 7.3 查看告警

```bash
GET /api/admin/alerts
Authorization: Bearer <token>
```

## 9. 安全设置

### 9.1 修改管理员密码

```bash
POST /api/admin/users/<用户名>
Authorization: Bearer <token>
Content-Type: application/json

{
    "password": "新密码"
}
```

### 9.2 启用 TLS 加密

1. 将证书文件放入应用数据目录
2. 通过 API 修改配置启用 TLS：
```bash
POST /api/admin/config
Authorization: Bearer <token>
Content-Type: application/json

{
    "tlsEnabled": true
}
```

> 注意：默认使用自签名证书，建议替换为正式证书。

### 9.3 暴力破解防护

系统内置暴力破解防护机制：
- 同一 IP 连续 5 次认证失败后，该 IP 被封禁 15 分钟
- 封禁期间的认证请求会直接被拒绝
- 封禁记录储存在内存中，服务器重启后重置

### 9.4 密码存储

用户密码使用 PBKDF2WithHmacSHA256 算法加盐哈希存储，迭代次数 120000。

### 9.5 数据加密

敏感字段支持 AES-256-GCM 加密存储，密钥由用户密码通过 PBKDF2 派生。

## 10. 性能调优

### 10.1 推荐配置

| 场景 | httpPort | maxConnections | idleTimeoutSeconds | tlsEnabled |
|------|---------|---------------|-------------------|------------|
| 开发测试 | 8080 | 50 | 300 | false |
| 生产环境(低负载) | 8080 | 100 | 600 | true |
| 生产环境(高负载) | 8080 | 200 | 300 | true |

### 10.2 SQLite 优化

系统已启用以下 SQLite 优化：
- **WAL 模式**: 读写并发，提升写入性能
- **mmap_size**: 256MB 内存映射，加速文件读取
- **cache_size**: 8000 页（约 32MB）缓存
- **temp_store**: MEMORY 模式，临时表存储在内存
- **page_size**: 4096 字节，适配闪存设备

### 10.3 缓存配置

系统使用 Caffeine 内存缓存：
- **文档缓存**: 最大 1000 条，自动淘汰
- **集合缓存**: 最大 100 条，更新时自动失效

## 11. 常见问题

### 11.1 无法连接到服务器

**可能的原因：**
1. 服务器未启动 — 检查应用界面是否显示「运行中」
2. IP 地址错误 — 确认设备 IP 地址（查看应用界面顶部）
3. 端口被占用 — 修改配置中的端口号
4. 防火墙阻挡 — 检查路由器/防火墙设置
5. 网络不在同一网段 — 确认客户端和服务器在同一个局域网

### 11.2 服务器自动停止

- 检查电池优化设置 — 在系统设置中将应用加入「不受电池优化」列表
- 检查省电模式 — 关闭系统省电模式
- 维持充电状态 — 建议连接充电器运行

### 11.3 性能不佳

- 使用有线网络（USB 转以太网）替代 WiFi
- 减少同时连接的客户端数量
- 定期清理不用的数据库和过期数据
- 避免执行无索引的全表扫描查询

### 11.4 数据库备份失败

- 检查存储空间是否充足
- 确认没有其他进程占用数据库文件
- 检查备份目录权限

### 11.5 忘记管理员密码

1. 停止服务器
2. 清除应用数据（注意：会删除所有数据）

### 12.3 TCP 客户端 (Java)

```java
import java.io.*;
import java.net.*;

public class MobileDBClient {
    private Socket socket;
    private DataInputStream in;
    private DataOutputStream out;

    public void connect(String host, int port) throws IOException {
        socket = new Socket(host, port);
        in = new DataInputStream(socket.getInputStream());
        out = new DataOutputStream(socket.getOutputStream());
    }

    public void sendRequest(String type, String jsonParams) throws IOException {
        String body = String.format("{\"type\":\"%s\",\"params\":%s}", type, jsonParams);
        out.writeInt(body.getBytes().length);
        out.write(body.getBytes());
        out.flush();
    }

    public String receiveResponse() throws IOException {
        int length = in.readInt();
        byte[] data = new byte[length];
        in.readFully(data);
        return new String(data);
    }

    public String authenticate(String username, String password) throws IOException {
        String params = String.format("{\"username\":\"%s\",\"password\":\"%s\"}", username, password);
        sendRequest("AUTH", params);
        return receiveResponse();
    }

    public void close() throws IOException {
        if (socket != null) socket.close();
    }
}
```

### 12.4 TCP 客户端 (JavaScript/Node.js)

```javascript
const net = require('net');

class MobileDBClient {
    constructor(host, port = 9090) {
        this.host = host;
        this.port = port;
        this.client = null;
    }

    connect() {
        return new Promise((resolve, reject) => {
            this.client = new net.Socket();
            this.client.connect(this.port, this.host, () => resolve());
            this.client.on('error', (err) => reject(err));
        });
    }

    sendRequest(type, params) {
        return new Promise((resolve, reject) => {
            const body = JSON.stringify({ type, params });
            const header = Buffer.alloc(4);
            header.writeUInt32BE(Buffer.byteLength(body));
            this.client.write(Buffer.concat([header, Buffer.from(body)]));

            this.client.once('data', (data) => {
                const length = data.readUInt32BE(0);
                const response = data.slice(4, 4 + length).toString();
                resolve(JSON.parse(response));
            });
        });
    }
}
```
