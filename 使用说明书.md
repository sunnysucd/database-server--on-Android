# AndroidDBServer 使用说明书

## 1. 概述

AndroidDBServer 是一款运行在 Android 设备上的多功能数据库服务器，提供 **TCP 协议**、**RESTful API** 和 **Web 管理面板**三种访问方式，支持远程 SQLite 数据库操作。

### 核心特性

- **三种访问方式**：TCP 协议（Netty）、REST API（Ktor）、Web 管理界面
- **多用户认证**：PBKDF2 + SHA-256 加密，Token 鉴权（24小时有效期）
- **安全防护**：IP 封禁（暴力破解防护）、速率限制、SQL 注入过滤
- **TLS 加密**：可选启用 TLS 1.2+ 加密通信
- **监控告警**：CPU/内存/存储/连接数实时监控，支持通知与 Webhook 告警
- **查询缓存**：LRU 缓存，自动失效
- **连接池**：支持并发连接限制、空闲超时断开
- **审计日志**：记录所有数据库操作
- **省电模式**：设备未充电时自动降低连接数
- **开机自启**：`BootReceiver` 监听设备启动自动运行

## 2. 安装与启动

### 2.1 安装 APK

```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

### 2.2 启动服务

1. 打开应用，点击 **START** 按钮
2. 状态指示灯变为绿色 "Running"
3. 服务自动启动三个服务器：
   - **TCP 服务器**：默认端口 `8899`
   - **REST API 服务器**：默认端口 `8080`
   - **Web 管理面板**：默认端口 `8081`

### 2.3 停止服务

点击 **STOP** 按钮，所有服务器将安全关闭。

> **注意**：服务以前台服务运行，通知栏会显示 "数据库服务器运行中"。Android 13+ 需授予通知权限。

## 3. 访问方式

### 3.1 Web 管理面板（推荐）

打开浏览器访问：

```
http://<设备IP>:8081/admin/
```

**默认登录凭据：**

| 用户名 | 密码 |
|--------|------|
| admin  | admin123 |

管理面板包含以下功能页（共 7 个标签页）：

| 标签页 | 功能 |
|--------|------|
| **Dashboard** | 概览：活跃连接数、最大连接数、数据库大小、表数量 |
| **SQL Terminal** | SQL 终端：执行任意 SELECT 查询，查看 JSON 结果 |
| **Users** | 用户管理：创建/删除用户，设置管理员权限 |
| **Tables** | 表管理：查看表结构(Schema)、删除表（系统表受保护） |
| **Backup** | 备份：触发数据库备份时间戳 |
| **Logs** | 审计日志：查看操作记录（时间、IP、用户、操作、状态） |
| **Connections** | 连接管理：查看当前活跃 TCP 连接详情 |

> **备份文件下载**：直接访问 `http://<设备IP>:8081/api/backup`（需携带 Bearer Token）可下载完整的 SQLite 数据库文件。

### 3.2 REST API

基础地址：

```
http://<设备IP>:8080
```

#### 认证

```bash
# 登录获取 Token
curl -X POST http://192.168.1.100:8080/api/auth \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin123"}'

# 响应示例
{"success": true, "message": "Authentication successful", "data": {"token": "abc123...", "username": "admin", "isAdmin": true}}
```

#### 健康检查

```bash
# Ping
curl http://192.168.1.100:8080/api/ping
# → {"success": true, "message": "pong"}

# Health
curl http://192.168.1.100:8080/health
# → {"status": "healthy"}
```

#### 数据库操作（需 Token）

```bash
# 查询
curl -X POST http://192.168.1.100:8080/api/query \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"sql": "SELECT * FROM _sys_users", "params": [], "limit": 10}'

# 插入
curl -X POST http://192.168.1.100:8080/api/insert \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"sql": "INSERT INTO my_table (name, value) VALUES (?, ?)", "params": ["foo", "bar"]}'

# 更新
curl -X POST http://192.168.1.100:8080/api/update \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"sql": "UPDATE my_table SET value = ? WHERE name = ?", "params": ["baz", "foo"]}'

# 删除
curl -X POST http://192.168.1.100:8080/api/delete \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"sql": "DELETE FROM my_table WHERE id = ?", "params": [1]}'

# 创建表
curl -X POST http://192.168.1.100:8080/api/create-table \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"sql": "CREATE TABLE IF NOT EXISTS my_table (id INTEGER PRIMARY KEY, name TEXT, value TEXT)"}'
```

#### 批量操作

```bash
curl -X POST http://192.168.1.100:8080/api/batch \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "operations": [
      {"type": "insert", "sql": "INSERT INTO my_table (name) VALUES (?)", "params": ["a"]},
      {"type": "insert", "sql": "INSERT INTO my_table (name) VALUES (?)", "params": ["b"]}
    ]
  }'
```
> 批量操作在一个事务中执行，任一操作失败则全部回滚。

#### 系统信息

```bash
# 表列表
curl -H "Authorization: Bearer <token>" http://192.168.1.100:8080/api/tables

# 数据库信息
curl -H "Authorization: Bearer <token>" http://192.168.1.100:8080/api/info

# 活跃连接
curl -H "Authorization: Bearer <token>" http://192.168.1.100:8080/api/connections

# 审计日志
curl -H "Authorization: Bearer <token>" "http://192.168.1.100:8080/api/audit-log?limit=50"

# 历史指标
curl -H "Authorization: Bearer <token>" "http://192.168.1.100:8080/api/metrics?hours=24"

# 当前指标
curl -H "Authorization: Bearer <token>" http://192.168.1.100:8080/api/metrics/current

# 系统日志
curl -H "Authorization: Bearer <token>" "http://192.168.1.100:8080/api/logs?lines=200"
```

#### REST API 错误码速查

| 错误码 | 含义 |
|--------|------|
| `AUTH_FAILED` | 用户名或密码错误 |
| `IP_BANNED` | IP 已被临时封禁（5 次失败尝试） |
| `INVALID_TOKEN` | Token 无效 |
| `TOKEN_EXPIRED` | Token 已过期（24小时） |
| `RATE_LIMITED` | 请求频率超限（200次/分钟） |
| `FORBIDDEN` | SQL 操作被安全策略禁止 |
| `QUERY_ERROR` | 查询执行失败 |
| `NOT_FOUND` | 端点不存在 |

### 3.3 TCP 协议

#### 连接

```bash
# 明文连接
nc <设备IP> 8899

# TLS 加密连接（需启用 TLS）
nc <设备IP> 8900
```

#### 协议格式

基于文本的 JSON 协议，每行一条 JSON 消息。

**请求格式：**

```json
{"action": "auth", "username": "admin", "password": "admin123"}
{"action": "ping"}
{"action": "query", "sql": "SELECT * FROM _sys_users", "params": [], "limit": 10}
{"action": "insert", "sql": "INSERT INTO my_table (name) VALUES (?)", "params": ["hello"]}
{"action": "update", "sql": "UPDATE my_table SET value = ? WHERE id = ?", "params": ["world", 1]}
{"action": "delete", "sql": "DELETE FROM my_table WHERE id = ?", "params": [1]}
{"action": "create_table", "sql": "CREATE TABLE IF NOT EXISTS my_table (id INTEGER PRIMARY KEY, name TEXT)"}
{"action": "tables"}
{"action": "info"}
{"action": "batch", "data": [{"action": "insert", "sql": "INSERT INTO my_table (name) VALUES (?)", "params": ["a"]}]}
```

**认证流程示例：**

```
→ {"action": "auth", "username": "admin", "password": "admin123"}
← {"success": true, "message": "Authentication successful", "data": {"token": "...", "username": "admin"}}
→ {"action": "query", "sql": "SELECT 1"}
← {"success": true, "message": "Query executed", "data": [{"1": 1}], "affectedRows": 1, "queryTime": 5}
```

> **注意**：TCP 连接需先发送 `auth` 命令完成认证，否则无法执行其他操作。未认证的连接会在空闲超时后自动断开。

#### TCP 选项

| 选项 | 默认值 |
|------|--------|
| 连接超时 | 300 秒（5 分钟） |
| 最大连接数 | 50 |
| SO_BACKLOG | 128 |
| SO_KEEPALIVE | 启用 |
| TCP_NODELAY | 启用 |
| 接收缓冲区 | 64 KB |
| 发送缓冲区 | 64 KB |

## 4. 配置说明

所有配置通过 `ServerConfig` 管理，持久化存储在 Android `SharedPreferences` 中。

### 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `port` | Int | 8899 | TCP 服务器端口 |
| `maxConnections` | Int | 50 | 最大并发 TCP 连接数 |
| `idleTimeoutMs` | Long | 300000 | 闲置连接超时（毫秒） |
| `cacheSize` | Int | 100 | 查询缓存最大条目数 |
| `mmapSize` | Long | 67108864 | SQLite 内存映射大小（字节） |
| `cachePageSize` | Int | 2000 | SQLite 页面缓存大小 |
| `enableTLS` | Boolean | false | 是否启用 TLS 加密 |
| `tlsPort` | Int | 8900 | TLS 端口 |
| `bruteForceThreshold` | Int | 5 | 封禁前允许的失败登录次数 |
| `bruteForceBanMinutes` | Int | 15 | IP 封禁时长（分钟） |
| `queryResultLimit` | Int | 1000 | 查询默认最大返回行数 |
| `logLevel` | String | "INFO" | 日志级别（VERBOSE/DEBUG/INFO/WARN/ERROR/FATAL） |

> 配置目前为固定值，可通过修改代码中的 `ServerConfig.kt` 调整。

## 5. SQL 安全策略

### 允许的操作

| 操作 | 说明 |
|------|------|
| `SELECT` | 查询数据 |
| `INSERT` | 插入数据 |
| `UPDATE` | 更新数据 |
| `DELETE` | 删除数据 |
| `CREATE TABLE` | 创建表 |
| `BEGIN / COMMIT / ROLLBACK` | 事务控制 |
| `SAVEPOINT / RELEASE` | 保存点 |

### 禁止的操作

| 操作 | 原因 |
|------|------|
| `DROP TABLE` | 破坏性操作，需通过专用 API |
| `ALTER TABLE` | 修改表结构 |
| `PRAGMA` | 修改数据库配置 |
| `VACUUM` | 数据库压缩 |
| `ATTACH / DETACH` | 附加外部数据库 |
| `CREATE / DROP INDEX / VIEW / TRIGGER` | 索引/视图/触发器操作 |
| SQL 注释（`--`、`/* */`） | 防止 SQL 注入 |
| 多语句（分号） | 防止 SQL 注入 |

### 安全设计

- **参数化查询**：所有 SQL 使用 `?` 占位符传递参数，杜绝拼接注入
- **Token 鉴权**：Bearer Token 有效期 24 小时，存储在 `_sys_users` 表
- **速率限制**：`/api/auth` 端点限制 200 次/分钟/IP
- **暴力破解防护**：5 次失败登录后封禁 IP 15 分钟
- **系统表保护**：`_sys_*` 表无法通过管理面板删除

## 6. 系统表说明

首次启动时自动创建以下系统表：

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `_sys_users` | 用户账户 | `id`, `username`, `password_hash`, `password_salt`, `token`, `token_expiry`, `is_admin`, `created_at` |

| `_sys_config` | 配置存储 | `key`, `value` |

## 7. 监控与告警

### 指标采集（每 5 秒）

| 指标 | 来源 | 说明 |
|------|------|------|
| CPU 使用率 | `/proc/stat` | 系统 CPU 占用百分比 |
| 内存使用率 | `ActivityManager` | 已用内存百分比 |
| 存储空间 | `StatFs` | 剩余/总存储（MB） |
| 活跃连接数 | `ConnectionPool` | 当前 TCP 连接数 |
| QPS | 审计日志 | 每秒查询数 |
| TPS | 审计日志 | 每秒事务数 |

### 告警阈值

| 告警类型 | 触发条件 | 通知优先级 |
|----------|----------|-----------|
| `CPU_HIGH` | CPU > 80% | HIGH |
| `MEMORY_HIGH` | 内存 > 90% | HIGH |
| `STORAGE_LOW` | 存储 < 500MB | DEFAULT |
| `CONNECTIONS_HIGH` | 连接数 > 80% 上限 | DEFAULT |

### 告警渠道

- **Android 通知**：通过 `NotificationManager` 推送
- **Webhook**：HTTP POST JSON 至配置的 URL
- **审计日志**：所有告警记录在审计日志中

## 8. TLS 加密

### 启用方式

修改 `ServerConfig.kt` 中 `enableTLS = true`（当前需代码修改）。

### 证书

- 优先从 `assets/server.p12` 加载 PKCS12 格式证书
- 回退自动生成自签名证书（RSA 2048位，1年有效期）
- 支持的密码套件：
  - `TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384`
  - `TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256`
  - `TLS_RSA_WITH_AES_256_GCM_SHA384`
  - `TLS_RSA_WITH_AES_128_GCM_SHA256`

### 连接方式

```bash
# TCP TLS
openssl s_client -connect <设备IP>:8900

# 使用自定义证书的客户端需信任自签名证书
```

## 9. 常见问题

### 9.1 无法连接服务器

1. 确认设备与应用在同一网络
2. 检查服务状态是否为绿色 "Running"
3. 确认端口未被防火墙阻挡
4. 使用 `adb logcat -s DatabaseService:I` 查看启动日志

### 9.2 浏览器访问报错

| 错误 | 原因 | 解决 |
|------|------|------|
| `ERR_CONNECTION_REFUSED` | 服务未启动或端口错误 | 确认 START 成功 |
| `ERR_INVALID_HTTP_RESPONSE` | 端口错误 | 使用正确端口（8080/8081） |
| `Endpoint not found` | 路径错误 | 使用 `/admin/` 或 `/api/ping` |

### 9.3 登录失败

- 默认凭据：`admin` / `admin123`
- 连续失败 5 次后 IP 被封禁 15 分钟
- 可在 `AuthService.kt` 中调整 `MAX_LOGIN_ATTEMPTS` 和 `BAN_DURATION_MS`

### 9.4 服务闪退

1. 检查项目路径是否包含中文字符——需在 `gradle.properties` 中设置 `android.overridePathCheck=true`
2. 使用 `adb logcat -s AndroidRuntime:E` 查看崩溃日志
3. 常见问题：Caffeine 缓存库不兼容 Android → 已替换为 LinkedHashMap 实现

### 9.5 端口冲突

默认端口占用情况：

| 端口 | 用途 |
|------|------|
| 8899 | TCP 服务器（明文） |
| 8900 | TCP 服务器（TLS） |
| 8080 | REST API 服务器 |
| 8081 | Web 管理面板 |

如需修改，编辑 `DatabaseService.kt` 中的端口参数。

## 11. 开发与构建

### 环境要求

- Android Studio Hedgehog (2023.1.1) 或更高
- JDK 17+
- Android SDK 34
- Gradle 8.2+

### 构建命令

```bash
# Debug APK
./gradlew assembleDebug

# Release APK（需配置签名）
./gradlew assembleRelease

# 安装到设备
adb install -r app/build/outputs/apk/debug/app-debug.apk

# 查看日志
adb logcat -s DatabaseService:I RESTAPIServer:I WebAdminServer:I
```

### 关键技术栈

| 技术 | 用途 |
|------|------|
| Kotlin 1.9.20 | 开发语言 |
| Ktor 2.3.9 | REST API + Web Admin 服务器 |
| Netty 4.1.108 | TCP 服务器 |
| SQLite + WAL | 数据库引擎 |
| BouncyCastle 1.78 | TLS 证书生成 |
| Gson 2.10.1 | JSON 序列化 |
| PBKDF2WithHmacSHA256 | 密码哈希 |
